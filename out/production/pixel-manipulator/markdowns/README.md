# Image Processor Final Project

This project is used to read, alter and write an image provided to the program. For this program I
created an image using chatGPT that had a lot of contrasting colors. I give myself permission to use
this image for this project.

Citation: OpenAI. “Image generated by ChatGPT based on user prompt.” 2025. AI-generated image.

This program consists of a model package which holds 4 packages that contain the functionality of
the program. Those packages are image, operations, read, and write.

# controller
This class sets up the input loop and handles and exceptions returned from bad command inputs by the user.
All exceptions are caught at one point and made as IllegalArgumentExceptions in the command classes. 
This is then caught by the controller formated by the messanger method and add to the appendable. 
The contents of teh appendable are then displayed to the user.

# commands
This class is used to hold the strategies behind command calls by the user for loading, altering and saving images.
The user will pass commands load, save or an operation command such as darken, lighten, red-component,
green-component, blue-component, luma, value or intensity. The user will additionally pass any other 
needed arguments such as file path, value, or image name. Once the command is called the action is performed.

### This package consists of:
- #### ICommands - Interface
  -     This is the interface for running all of the commands passed.
- #### CommandMap - Implementation
  -     This is the class creates and returns to the controller the map of all commands available to the user.
- #### CommandsImpl - Implementation
  -     This class holds the main constructor for all commands.
- #### commands.load - Package
  -     Holds command classes for loading images.
- #### commands.operations - Package
  -     Holds command classes for altering images.
- #### commands.save - Package
  -     Holds command classes for saving images.

## commands.operations
This package contains all the calls to the operations from model.operations. Each command takes in the 
image desired to be altered which is retried from storage using the image name provided by the user. The 
command then performs the alteration on the image. The image is then stored using a new image name provided
by the user and removes the old image from the storage.
Note the Darken and Brighten commands additionally take a value. This value the amount of darkening or 
lightening the user desires the image to have.

the user.

### This package consists of:

# Model
This package contains the main functionality of the program. This package contains four packages that each 
hold the specified actions logic.

## model.read

The read package contains the interface and implementation of the read interface. This impl constructs
an image reader that takes a Readable. The logic behind using a readable is so that future expansion
of this project can use other filepath types. The impl its self converts a Readable to a string.
It then reads and parses the image from the images filepath.

### This package consists of:
- #### IImageReader - Interface
  -     This class is the interface for the read opertations.
- #### PPMImageReader - Implementation
  -      This class is the implementation of the IImageReader for PPM image files. This is used to
         read image files that have the .ppm extension.  
- #### ReadBinaryImage - Implementation
  -      This class is the implementation of the IImage reader for binary image image files. This is used to
         read image files that have the .jpeg, .png, .bmp extensions.  


## model.write:
The read package contains the interface and implementation of the write interface. The impl constructs
a new write object. This takes the desired name and operation to create a file name. The writer
takes the image and converts the images pixels into a List of strings. The writer then appends
it to the head of the file and writes it.

### This package consists of:

- #### IImageWriter - Interface
    -     This class is the interface for the write opertations.
- #### PPMImageWriter - Implementation
    -     This class is the implementation of teh ImageWriter for PPM image files. This is used to write
          image files that use the .ppm extension to be written.
- #### WriteBinaryImage - Abstract Implementation
    -     This class is the implementation of teh ImageWriter for binary image files. This is used to write
          image files that use the .jpeg, .bmp, .png extensions to be written.
- #### BMPImageWriter - Strategy
    -     This class is used to hold the extension being called in the abstract WriteBinaryImage class.
- #### JPEGImageWriter - Strategy
    -     This class is used to hold the extension being called in the abstract WriteBinaryImage class.
- #### PNGImageWriter - Strategy
    -     This class is used to hold the extension being called in the abstract WriteBinaryImage class.


## model.image:

The image package contains interfaces for the Pixel and Image.
The image implementation is where the image as a whole is stored and accessed from. It holds the 
height, width and max value of the image in its fields. The image impl also sets the pixels to the 
correct location of the image.
The pixel implementation is used to hold and access the individual values for rgb. The impl also
sets the values for the pixel based on values provided by the operations.

### This package consists of:

- #### IImage - Interface
    -     This class is the interface which stores the array of pixels.

- #### IPixel - Interface
    -     This class is the interface which stores th individual pixels.
- #### ImageImpl - Implementation
    -     This class is the implementation of the IImage interface. This class is used to store the 
          image pixels in and array and store the height width and max value data of the image.
- #### Pixel - Implementation
    -     This class is the implementation of the IPixel interface. This is used to store and retrieve
          the individual rgb values of the the pixels as a pixel.



## model.operations:
The operations package holds an operations interface that applies the operation to the pixel.
The implementation for the operations interface is split between a Abstract class for all operations
and an abstract class for Greyscale operations which extends the Abstract Operations class. The 
reasons behind splitting up in this way is because all classes share the core functionality of the
apply method which iterates through each pixel and sets to a new value. The reason for a 
grey scale abstract class is because the grey scale operations also share some key functionality 
in that there is a new value being generated and applied to the pixels.
The implementation of the greyscale classes hold just the calculations for the new rgb value. 
The brighten, darken and abstract greyscale are all used to set how the values will be applied to 
the pixels.

### This package consists of:
- #### IOperations - Interface
    -     This class is the interface for altering the images provided and providing a new image in its place.
- #### AbstractOperations - Abstract Implementation
    -     This class is an abstract implementation of all operation types.   
- #### BrightenOperation - Implementation 
  -       This class contains the logic for the brighten strategy that is called by the AbstractOperations class.
- #### DarkenOperation - Implementation
  -       This class contains the logic for the darken strategy that is called by the abstract class.
- #### GreyScaleAbstract - Abstract Implementation
  -     This class is an abstract implementation of all grey scale operation types.  
- #### GreyScaleOperation - Implementation
  -       This class contains the main logic for all greyscale strategies that is called by the AbstractOperations class. 
          This creates luma, intencity, value, red, green, and blue grey scale images.
- #### LumaOperation - Implementation
  -       This class contains the diffencial logic for the luma grey scale that is called by the GreyscaleAbstract class.
- #### IntensityOperation - Implementation
  -       This class contains the diffencial logic for the intensity grey scale that is called by the GreyscaleAbstract class.
- #### ValueOperation - Implementation
  -       This class contains the diffencial logic for the value grey scale that is called by the GreyscaleAbstract class.


## VIEW
The view is used do create a GUI that allows for the user to interact with the program by selecting the operations the pressing 
the desired action button. The view is split into 2 packages; Panels and Actions.
The reason for the splitting of the view into multiple classes is to make the code easier to read and adjust.
The panel classes will create the panel and layout of the GUI. Each functional panel has its own class and 
an abstract class is used to reduce code reuse. The action classes consist of the different listener actions.
There is an abstract class that is used to reduce repeated code within the action methods. Additionally, there
is an action runner class. This class is used to run the model. The run method within this class creates a new
scanner and adds the instructions from the GUI fields to that scanner. The operation key which is also passed
from the GUI the apply load or save button is then used to call the desired command. 

Conclusion:
The reasons behind the abstraction is to make it as easy as possible to expand this project for part
2, and 3 if need be. Additionally, the abstraction ws used to reduce code reuse and for readability.
